希望和WeHelp訓練營的大家一起學習扎實的硬實力

從頂尖建築師到數據顧問，再到後端開發。
我有工程邏輯與商業洞察背景，進一步加強技術面，準備好跨出轉職後端工程師的最後一哩路。

1. 個人簡介和申請動機

個人簡介

我是一名擁有跨領域背景的後端工程師。我曾於荷蘭與台灣擔任建築師及不動產顧問，隨後轉向數據領域，於荷蘭 HippoLine 擔任初階數據智能顧問。

在過去 6 個月裡，我全職投入後端開發的學習，將過去在代爾夫特理工大學 (TU Delft) 培養的嚴謹工程邏輯應用於程式設計中，並實踐上線專案。我的核心特點是在荷蘭工作超過5年，有建築與數據分析經驗，實際的工作軟實力希望將程式的硬實力補齊。

申請動機

在過去半年的自學期間，我盡可能地運用資源並完成了多個專案。雖然學習到很多，但我知道自己距離成為真正符合業界標準的工程師仍有一段距離。自學過程中，我最缺乏的是深度的同儕討論以及業界資深人士的 Code Review。

我非常渴望能加入 WeHelp 訓練營。WeHelp 任務導向、職業導向且嚴格扎實的訓練模式，正是我所需要的。我希望在訓練營結束後，能帶著扎實的硬實力和自信，成功跨出轉職的最後一步。

2. 曾經做過哪些軟體工程技術相關的學習？

在過去半年的密集學習中，我完成了兩項後端專案到上線的全流程，目前的瓶頸是不知道如何再往深入做：

技術棧 (Tech Stack)
  • 後端與框架： Python3, FastAPI, Pydantic, RESTful API, asyncio
  • 資料庫與 ORM： PostgreSQL, SQLite, SQLAlchemy, SQLModel, Redis, Celery
  • DevOps 與工具： Git, Linux, Docker, AWS (EC2, S3), CI/CD (GitHub Actions)
實作專案 (Projects)
  • Project-LINKER： 使用 FastAPI 與 PostgreSQL 開發的系統，實作 RBAC 與 JWT 進行安全存取控制。
  • Project-PayFlow-API： 建立事件驅動的支付 API，串接 ECPay Webhooks；並透過排他鎖 (Exclusive Locks) 與 asyncio，確保高併發下的數據一致性。

3. 如果參與這個訓練，會怎麼安排學習時間？

目前我是全職準備轉職的狀態。在訓練營期間，我將以「全職工作」的態度面對，每天投入至少 8 小時進行學習、實作與同儕討論。

我非常重視學習效率與續航力，因此在全力衝刺的同時，也會維持規律運動與充足睡眠。若在訓練營後期提前找到工作，我也承諾會在不影響正職的前提下，利用所有閒暇時間，全心全意地將訓練營的專案扎實完成。

4. 請簡述對於軟體工程的領域偏好和職涯想像

相比於沒有正確答案的建築理念，我對有衡量標準的技術性的工作內容有興趣，像「後端開發」與「系統設計」有正確答案又沒有唯一答案的工作，此外，我喜歡將東西從無到有做出來的成就感。

對於未來的職涯想像，我的日常工作不僅是修改 Code、設計架構、串接 API 與測試，之後一系列長期維持服務器和code、debug等。也有與 PM 溝通釐清需求，與同事透過 GitHub 和 Code Review 進行深度對話，並具備對 AI 回答進行判斷的能力。我期許自己成為一位具備產品思維，且言之有物、硬實力扎實的工程師。

5. 請描述一件產生明顯負面情緒的經歷，如何處理該情緒？

我在歐洲攻讀碩士畢業時，是COVID疫情大隔離。當時面對碩士論文的 DEADLINE，我發現自己前期的研究方向不對，導致很多部分無法完成。當時的困境和焦慮是，我無法交出一份不及格的作品(若延畢將對家裡造成很大經濟負擔)。

那種「努力到身心俱疲卻看不到盡頭」的恐懼與不確定感，我產生了嚴重的逃避心理。我當時只能靠一直睡覺來逃避，醒著時伴隨劇烈頭痛，躺下則是不斷的噩夢。最後，我勉強完成論文並準時畢業。

這段經歷帶給我的成長： 後來當我再次遇到類似情況，或是情緒卡關無法前進時，我較可以處理。我會先停下來釐清情緒，思考甚麼是現在可以做的並切小一步一步完成或是以優先級去完成，同時透過運動來重置大腦狀態，以更健康的韌性去面對壓力。

6. 關於這份申請網頁，分享一個開發時的技術心得

在 AI 時代，常常聽到「大腦外包」的說法，但這次動手寫網頁，我喜歡這種邊做邊可以思考的節奏。

這就像我高中時最喜歡也最自豪的數學和物理一樣。看懂解答是一回事，但看完後一定要親自動手寫，那些知識與技術才會真正成為我自己的。這份踏實感是我熱愛寫程式的重要原因。

7. 如何看待人工智慧的發展對軟體工程師的影響？

為什麼在這個 AI 時代，我還想當工程師？

因為興趣。我喜歡將東西做出來，也喜歡技術導向、言之有物的東西。

從建築業轉行到軟體業，我深刻體會到兩者的不同。在建築業擔任到頂尖職位時，我面臨了傳統產業薪資天花板，且工作內容常常流於主觀討論或是政治化的決定。而在軟體工程的世界裡，技術是客觀的。

AI 與工程師的關係

我認為 AI 就像是一艘強大的電動船，但人才是最終的掌舵者。會開船的人，永遠比不會開船的人走得更遠。

AI 是基於人類的大數據來學習與運作，因此我們更需要理解並判斷 AI 產出的內容。當需要判斷的問題複雜到一定程度時，唯有具備扎實手刻經驗與能推導深層邏輯的工程師，才能真正解決問題。這也是為什麼在這個 AI 時代，我依然堅定要成為一名掌握硬實力、擁有產品思維的軟體工程師。

8. 從上次提出申請至今，多做了哪些努力？

這是我第一次申請 WeHelp。在知道這個計畫之前，我已經完全認同這樣的模式，並且在嘗試自學時，就是以「專案為導向」在穩健前進。我所做的努力包含：

  • 技術深耕： 限縮精力花在刀口上，專注學習 Python 與 FastAPI，並開始練習演算法題目。
  • 專案實戰： 從選題、定位開發分配任務到做出成品，一步步完成了三個專案，其中兩個已上線。
  • 解決問題： 遇到不會的問題，我會透過查閱技術文件、StackOverflow、IT邦、Udemy 課程或詢問 AI 來尋找解答。

然而，我也意識到目前的自學多半是套用現成模板，或是單向與 AI 討論，缺乏深度的架構思考與同儕交流。我已經做足了前期準備，現在需要訓練營的環境來更扎實、更進一步。

9. 其他想要對我們說的事情

我想分享我一路走來，不斷追求突破與卓越的職涯軌跡。我非常希望自己能照顧好自己，並有能力照亮別人：

  • 追求卓越的學習力： 畢業於 QS 世界排名前三的代爾夫特理工大學 (TU Delft) 建築碩士。在學期間領取獎學金、參與社團並擔任家教，也曾獨自闖入建築競賽前八強。在過往的工作中，為公司得標（第一名）也是我的日常。
  • 跨越國界與領域的韌性： 通過錄取率僅 5% 的台灣建築師高考，並在畢業後成功留在非歐盟人士極難留下的荷蘭工作。用非母語完成專業認證，僅花三年便取得荷蘭建築師執照（通常需 3-5 年）。
  • 擁抱數據與科技： 在工作之餘學習荷蘭語，完成 IBM 數據分析認證並進入 BI 產業工作。隨後自學後端開發，半年內完成並上線兩個具備商業潛力的專案。寫程式與 Data-Driven 是我喜歡的，也是市場所需的。

可以看到我的職涯一直在追求突破，穩健扎實。我在多年工作經驗中探索並扎實前進。我非常期待能將這份充滿正能量的生涯態度帶入 WeHelp。
